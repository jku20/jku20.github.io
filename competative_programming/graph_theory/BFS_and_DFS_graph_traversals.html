Graph Traversals

These are algorithms which visit every node in a graph in a certain order.

Uses

  • checking connectivity
  • finding cycles This can also be done through counting edges in a component. If there are no cycles in a component of nodes, than it contains exactly edges.
  • finding and retrieving cycles It can also be done in a way which allows reconstruction. Initially color all vertices white. From each unvisited vertex start a DFS. When entering a vertex mark it gray and when exiting mark it black. If during the DFS a gray vertex is found than a cycle has been found. Keeping an array with the parent of each vertex allows the cycle to be retrieved.1 Note if the graph is undirected ignore the edge between a node and its parent.
  • bipartite check using a DFS, coloring initial blue and neighbors opposite colors and checking is possible. Does NOT work in the general case of colors, that problem is NP-hard.

Depth First Search (DFS)

DFS works through searching down a path until it reaches a node on the path which does not have any unvisited neighbors. It will than go one node back in the path and recursively search, going through all of its neighbors. The search terminates when all edges and nodes have been visited. It works in , being number of nodes and being number of edges.

Implementation

Recursively: simple to understand. This is using an implicit stack, the one which drives function calls and more!

//graph as adjacency list
vector<int> adj[N];
//input things to graph
// array maintained by the function
bool visited[N];

void dfs(int s) {
    if(visited[s]) return;
    visited[s] = true;

    //prosses node s
    
    for(auto u : adj[s])
    	dfs(u);
}

dfs(adj[i]); //where i is the wanted starting node

Iteratively: This works with a stack which simulates the call stack and recursion. This technique is called using an explicit stack.

//graph as adjacency list
vector<int> adj[N];
//input things to graph
//maintained array initially all set to false
bool visited[N];

stack<int> call;
call.push(i); //where i is the wanted starting node
while(call.size()) {
    int s = call.top(); call.pop();
    if(visisted[s]) continue;
    visited[s] = true;
	
    //proccess node s
        
    for(auto u : adj[s])
    call.push(u)
}

Choosing to use the iterative or recursive version of a DFS comes down to personal preference.

Breath First Search (BFS)

This algorithm processes the nodes, starting with all neighbors to the starting node. It will than do the same for all of those neighbors. It will continue until all nodes are processed, and than terminate. It runs in time, being number of nodes and being number of edges. This uses notably more memory than a DFS.

Implementation

Queue Implementation: It just adds the nodes to a queue in the correct order and than processes them. Almost like the queue version of the iterative method. 

vector<int> adj[N];

queue<int> q;
bool visited[N];
int distance[N]; //calculating distances from the starting node because why not

//vertex i is the starting vertex
visited[i] = true;
distance[i] = 0;
q.push(i);

while(!q.empty()) {
    int s = q.front(); q.pop();
    if(visited[s]) continue
    visited[s] = true;
	
    //prosses s
    
    for(auto u : adj[s]) {
        q.push(u);
        distance[u] = distance[s]+1;
    }
}

DFS with Iterative Deepening (IDDFS or IDS)

This is functionally similar to a BFS but uses the space of a DFS, linear, at the cost of time, however, it runs no more than twice as slow as a BFS.

IDS is used as an alternative to BFS in a large search tree. It runs in time and using space where is the branches from a node and is the depth of answer.

IDS works using two functions, one iterating over all depths and another which runs a depth-limited DFS (DLS), meaning, a DFS which is only allowed to run to a certain depth.

The iterator will iterate from 0 to , the maximum depth to search, and for each run a DLS to depth . It will use the result of the DLS to determine if it should continue the iterations or return the value from the DLS.

A DLS works like a DFS except it contains a counter for depth. When the DLS reaches a certain depth, it will return a specified value, possibly null or the current value, with a flag telling the program to either stop or continue the IDS.
