DP

Using an array or some other structure to store data to later be used. Find a recurrence relation and figure out if there is data you can store which is used more than once. Use memoization to store stuff. Do it recursively if iteratively is hard, but normally it shouldn't be too bad.

general uses (not exhaustive or perfect list)

  • finding an optimal solution
  • counting the number of solutions

look for

  • turn an exponential into something else
  • the current state is only dependent on the previous state(s)
  • is casework possible for subproblems
  • can you represent you data in a grid and find a path through the grid
  • can you look at all different states, do those states repeat
  • can you find a value which wants to be minimized or maximized (or both!)
  • can you combine turns in a turn based game so so don't need a case for both players?
  • is the dp array sorted? Can you use a binary search?
  • is compression possible?
