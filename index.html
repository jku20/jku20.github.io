Dijkstra's Algorithm

This finds the shortest path between two points using a greedy algorithm which works in a graph with NO NEGATIVE WEIGHTS.

This works similarly to SPFA but with a little bit more intelligence. Dijkstra's Algorithm starts with a starting node which has distance from the starting node. It than chooses the unvisited node with shortest distance from the start node and corrects its distance. A priority queue is normally used to keep this order. These distances are final and correct. This repeats until either all nodes are visited or the wanted node is reached (if you make that the case).

Dijkstra's goes through all edges once and adds each vertex to the priority queue once. This makes it for , .
